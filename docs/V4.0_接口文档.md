# 面型及 Rxy 分析工具 V4.0 接口文档

## 1. 版本与兼容性

- 接口版本：`V4.0`
- 功能基线：GitHub `main` 的 `V3.9.3`，提交 `c86647e`
- GUI 原有 XYZ/白光 DAT/Zeiss 文本/高度矩阵、ROI、多层扣减、平行度、Recipe、报告与批量流程保持原口径。
- V3.9.3 单文件保留不动；V4.0 入口改为 Python 包 `surface_analyzer`。

重要坐标约定：输入统一按右手坐标系解释，默认 X 向右、Y 向上、Z 向上。计算符号继续沿用主控页：

- `Rx = atan(+dZ/dY)`，单位 `µrad`
- `Ry = atan(-dZ/dX)`，单位 `µrad`
- 平行度差值：`测量面 - 基准面`

## 2. 本地启动接口

双击：

```text
start_surface_analyzer_v4_0.bat
```

检查环境但不打开窗口：

```bat
start_surface_analyzer_v4_0.bat --check
```

启动 GUI 并自动载入一个文件：

```bat
start_surface_analyzer_v4_0.bat --input "D:\data\sample.dat"
```

## 3. 无界面命令行接口

适合 C# 平台在扫描完成后启动一个 Python/EXE 进程，等待分析结束后读取 JSON。

```bat
.venv\Scripts\python.exe -m surface_analyzer ^
  --headless ^
  --input "D:\data\sample.xyz" ^
  --output-json "D:\result\sample_result.json" ^
  --x-column 0 --y-column 1 --z-column 2 ^
  --x-unit mm --y-unit mm --z-unit um ^
  --filter sigma_clip --sigma-k 3 --sigma-iterations 5 ^
  --max-points 100000
```

列索引从 `0` 开始。常规无表头 XYZ 即 `0/1/2`。文本接口可读取逗号、分号、Tab 或空格分隔，并会跳过前部多行参数说明，直到找到连续 XYZ 数值区。

参数：

| 参数 | 含义 | 默认值 |
|---|---|---:|
| `--input` | 输入文件绝对路径 | 必填 |
| `--output-json` | JSON 输出路径；省略则输出到控制台 | 空 |
| `--x-column/y-column/z-column` | 列名或 0 基列索引 | `0/1/2` |
| `--x-unit/y-unit/z-unit` | `mm`、`um`、`µm`、`nm` | `mm` |
| `--filter` | `off`、`mad`、`local_median`、`sigma_clip` | `off` |
| `--neighbor-k` | 局部中位数邻居数 | `12` |
| `--threshold-um` | 局部滤波阈值 | `5.0` |
| `--sigma-k` | 迭代 Sigma 倍数 | `3.0` |
| `--sigma-iterations` | Sigma 最大迭代次数 | `5` |
| `--max-points` | 大文本接口最大参与点数 | `100000` |
| `--transform` | 可重复传入姿态动作 | 空 |

姿态动作取值：`CW90`、`CCW90`、`ROT180`、`SWAP`、`FLIPX`、`FLIPY`、`ORIGIN(0,0)`。

## 4. JSON 输出

成功示例：

```json
{
  "version": "V4.0",
  "source": "sample.xyz",
  "input_points": 10000,
  "finite_points": 9998,
  "roi_points": 9998,
  "fitted_points": 9980,
  "filtered_points": 18,
  "sampled": false,
  "import_strategy": "文本全量读取",
  "transform_pipeline": [],
  "filter": {"mode": "sigma_clip"},
  "metrics": {
    "a": 0.00001,
    "b": -0.00002,
    "c": 1.2,
    "mean_z": 1.2,
    "rms": 0.4,
    "pv": 2.1,
    "ttv": 30.0,
    "rx": -20.0,
    "ry": -10.0
  },
  "warnings": [],
  "centroid": {"x_mm": 4.0, "y_mm": 5.0}
}
```

`mean_z/c/step_height` 为 mm；`rms/pv/ttv` 为 µm；`rx/ry/parallelism` 为 µrad。

退出码：

| 退出码 | 含义 |
|---:|---|
| `0` | 成功 |
| `2` | 参数错误 |
| `10` | 文件不存在、格式或列映射错误 |
| `20` | 分析计算失败 |
| `30` | JSON 输出失败 |

## 5. C# 调用示例

```csharp
using System.Diagnostics;

var psi = new ProcessStartInfo
{
    FileName = @"D:\claude\Projects\面型及Rxy分析工具\.venv\Scripts\python.exe",
    WorkingDirectory = @"D:\claude\Projects\面型及Rxy分析工具",
    UseShellExecute = false,
    RedirectStandardOutput = true,
    RedirectStandardError = true,
    CreateNoWindow = true,
};

psi.ArgumentList.Add("-m");
psi.ArgumentList.Add("surface_analyzer");
psi.ArgumentList.Add("--headless");
psi.ArgumentList.Add("--input");
psi.ArgumentList.Add(inputPath);
psi.ArgumentList.Add("--output-json");
psi.ArgumentList.Add(resultJsonPath);
psi.ArgumentList.Add("--z-unit");
psi.ArgumentList.Add("um");

using var process = Process.Start(psi)!;
await process.WaitForExitAsync();
var stdErr = await process.StandardError.ReadToEndAsync();
if (process.ExitCode != 0)
    throw new InvalidOperationException($"分析失败({process.ExitCode}): {stdErr}");
```

平台正式集成时建议传绝对路径，并为每次扫描建立独立输出目录，避免并行任务覆盖同名 JSON。

## 6. Python API

内存数组调用：

```python
from surface_analyzer import AnalysisOptions, analyze_xyz

options = AnalysisOptions(z_unit="um", filter_mode="sigma_clip")
result = analyze_xyz(x, y, z, options=options, source="scanner-001")
payload = result.to_dict()
```

文件调用：

```python
from surface_analyzer import AnalysisOptions, analyze_file

result = analyze_file(
    r"D:\data\sample.dat",
    options=AnalysisOptions(z_unit="um"),
    x_column=0,
    y_column=1,
    z_column=2,
    max_points=100000,
)
```

双面结果比较：

```python
from surface_analyzer import compare_plane_results

parallel = compare_plane_results(base_result, measure_result)
```

## 7. 当前接口边界

- GUI 仍是完整功能入口，包括 VR/基恩士高度矩阵、智能抓面 ROI、Recipe、报告和多层扣减。
- V4.0 首版无界面文件接口针对常规三列 XYZ/CSV/TXT/DAT/ASC 与 Excel；高度矩阵应先走 GUI，后续可单独扩展 `file_io.py`，不需要改 UI。
- 超过 64 MB 的无界面文本默认按文件位置流式抽样到 `--max-points`，不会全量加载到内存；JSON 会标记 `sampled=true`。
- 对降采样文件，拟合趋势通常稳定，但 PV/TTV 极值取决于抽样是否保留极值；需要最终计量结论时应使用 GUI 的空间网格 Min/Max 策略或增加接口策略。

## 8. Recipe 手动删除重放

V4.0 Recipe `schema_version=2` 可以保存手动框选删除操作。每条操作记录框选视图、坐标范围、当时的坐标变换、原始/去倾斜显示模式、拟合平面系数、ROI/滤波快照和源文件 SHA-256。

重新导入时仅在以下条件全部满足后重放：源文件 SHA-256 一致、导入/抽样签名一致、每次操作重算点数与原记录一致。任一校验失败都会取消全部手动删除重放，避免对错误文件或不同采样结果误删点。
