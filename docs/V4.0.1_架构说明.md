# V4.0.1 模块架构说明

V4.0.1 延续 V4.0 的模块化结构。V3.9.3 旧单文件已移入 `archive/legacy_versions/v3_single_file`，仅作为历史基线。

## 模块职责

| 文件 | 职责 | 常见修改场景 |
|---|---|---|
| `surface_analyzer/app.py` | 主窗口、左栏与右侧视图组织、主分析刷新流程 | 调整主页面 UI、信号连接、视图布局 |
| `surface_analyzer/widgets.py` | 无滚轮输入框、四视图、平行度和扣减诊断画布 | 调整画布样式和通用控件 |
| `surface_analyzer/mixins/analysis.py` | 平面拟合、滤波、坐标变换、Rx/Ry/PV/TTV | 修改计算公式或滤波算法 |
| `surface_analyzer/mixins/data_io.py` | GUI 文件导入、大文件策略、高度矩阵、列映射 | 增加设备格式或抽样策略 |
| `surface_analyzer/mixins/roi.py` | 矩形/圆形/智能抓面 ROI 与鼠标交互 | 修改抓面算法或 ROI 行为 |
| `surface_analyzer/mixins/gap.py` | 多层胶厚寄存器、匹配和扣减 | 修改多层扣减逻辑 |
| `surface_analyzer/mixins/parallelism.py` | 基准面/测量面、平行度和台阶报告 | 修改双面比较功能 |
| `surface_analyzer/mixins/recipe.py` | Recipe 序列化和恢复 | 增加可保存参数 |
| `surface_analyzer/mixins/reporting.py` | 主报告、CSV、批量处理 | 修改导出内容或报告排版 |
| `surface_analyzer/api.py` | 无界面 Python 公共接口和结果模型 | 增加平台可调用的计算能力 |
| `surface_analyzer/file_io.py` | 无界面 XYZ 文件读取 | 扩展 C# 接口支持的文件格式 |
| `surface_analyzer/cli.py` | GUI/无界面命令行入口和退出码 | 增加平台进程参数 |
| `surface_analyzer/config.py` | 默认值与大文件预设 | 调整全局默认策略 |
| `surface_analyzer/version.py` | 版本和源基线 | 发布版本时只改这里 |

## 依赖方向

```text
启动文件 / C#
      |
      v
cli.py ----------> api.py ----------> analysis.py
  |                  |
  v                  v
app.py             file_io.py
  |
  +--> widgets.py
  +--> data_io.py
  +--> roi.py
  +--> gap.py
  +--> parallelism.py
  +--> recipe.py
  +--> reporting.py
```

业务模块不反向导入 `app.py`，因此后续修改某个算法或导出模块时，不需要改窗口主类。`api.py` 不创建 Qt 窗口，可由 C# 进程或其他 Python 程序独立调用。

## 维护规则

1. 计算口径统一放在 `analysis.py`，GUI 和接口不要复制拟合公式。
2. 新设备文件格式优先放到 `data_io.py`；需要平台无界面调用时，再给 `file_io.py` 增加对应适配器。
3. 新增 Recipe 字段时保持向后兼容，读取旧 Recipe 必须有默认值。
4. 报告只消费分析结果，不在报告模块内重新计算指标。
5. 历史文件统一保存在 `archive/legacy_versions`，根目录只放当前版本入口。

## 自动检查

```bat
.venv\Scripts\python.exe -m unittest tests.test_v4_api -v
start_surface_analyzer_v4_0_1.bat --check
```

`archive/legacy_versions/v4_0_preview/tools/build_v4_modules.py` 是从 V3.9.3 拆分方法体的历史迁移脚本，不是日常启动依赖，不应再用它覆盖已经修改过的模块。
