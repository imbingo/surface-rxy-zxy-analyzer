# V4.0.2 本地测试清单

1. 双击 `start_surface_analyzer_v4_0_2.bat`，标题应显示 `V4.0.2`。
2. 导入一个过去能正常分析的小 XYZ/DAT，核对 Rx、Ry、PV、TTV 与 V3.9.3 一致。
3. 导入一个 Zeiss 大文件，核对默认文件位置抽样、导入点数和流畅度。
4. 导入白光分号 DAT 与带多行参数头的 XYZ，核对列映射和单位。
5. 分别测试矩形、圆形、智能抓面 ROI，清除当前/全部后视图应恢复。
6. 测试局部中位数和 Sigma 滤波，并导出主控报告图。
7. 测试多层扣减匹配诊断图。
8. 写入基准面和测量面，核对平行度与台阶结果、报告排版。
9. 导出再导入 Recipe，核对 ROI、滤波、大文件策略恢复。
10. 在命令行运行 `start_surface_analyzer_v4_0_2.bat --check`。
11. 用两个相隔的共面区域测试“同平面抓取”，确认只保留种子所在连通区域。
12. 模拟文件位置抽样，确认主指标显示 `≈`，报告和CSV写明“极值未保留，不可直接用于产线放行”。
13. 在 XY/XZ/YZ 各做一次手动框选删除并导出 Recipe，重新载入同一文件后确认删除点数完全一致。
14. 修改源文件任意一个字节后再次导入 Recipe，确认 SHA-256 校验阻止手动删除重放。

15. 导入含多行描述、数值参数、顶部列坐标、左侧行号和尾随空列的高度矩阵，核对自动起始行和矩阵尺寸。
16. 将“矩阵数据起始行”设为已知行号后重新导入，并确认 Recipe 能保存和恢复该设置。
17. 在 `1180×700` 和全屏窗口分别检查主控 XY、平行度双 3D，确认 XY 几何比例不随窗口改变。
18. 导出主控和平行度报告图，确认长条面保持 X/Y 比例，平行度结果卡文字无重叠。

需要回退时，按 `archive/legacy_versions/README.md` 中的提交号创建独立 Git worktree，不要覆盖根目录的 V4.0.2。
